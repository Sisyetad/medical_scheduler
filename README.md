# Project Title - Medical_Scheduler_Mobile

## Description

It is a system enabling patients to book appointments with doctors, managed efficiently by receptionists. It streamlines scheduling, updates, and coordination between patients and doctors.

## Features

### 1. Authentication and Authorization

- **User Registration and Login:** Users can sign up and log in using their email accounts.
- **Logout:** Users can securely log out of their accounts.
- **Role-Based Authorization:** The system supports different roles, ensuring that admins have higher privileges compared to regular users.

### 2. Business Features and CRUD Operations

#### **Patient History Management**

- **Create:** Doctors can add new patient history, including general information and diagnoses.
- **Read:** Doctors can access previous patient records to understand medical history and provide better treatment.
- **Update:** If incorrect information is found, doctors can request modifications, which will be reviewed and approved by the branch office.
- **Delete:** Patients can request the deletion of their information, and doctors can process these requests accordingly.

#### **Doctor Management**

- **Create:** Branch administrators can add new doctors to the hospital system.
- **Read:** Branch administrators can view detailed profiles of doctors, including their specialties and credentials.
- **Update:** Doctors can update specific parts of their profiles, such as contact details and work schedules.
- **Delete:** Branch administrators can suspend or remove doctors from the system if necessary.
